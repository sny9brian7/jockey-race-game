# ジョッキー視点レースゲーム

一人称（騎手視点）の3D競馬レースゲーム。ブラウザで動く静的ファイル構成。

## 構成
- `index.html` — 画面・HUD・オーバーレイ（タイトル/出馬表/カウントダウン/結果）
- `game.js` — ゲーム本体（コース生成、馬AI、レースロジック、描画）

## 実行方法
`index.html` をブラウザで開くだけ（ビルド不要）。

## 依存
- Three.js r128 — CDN（cdnjs）から読み込み。初回はネット接続が必要。

## 開発フロー（試作 → 反映 → 公開）
このフォルダ直下（`index.html` / `game.js` / `CLAUDE.md`）が「本番＝公開版」。
実験的な変更はここを直接いじらず、必ず `draft/` サブフォルダの中で試作する。

1. **試作**: `draft/index.html` / `draft/game.js` を編集して調整・実装する。
   `draft/` は `.gitignore` で除外済みなので、いくら試しても本番やGitに影響しない
2. **検証**: バランスに関わる変更は、実物の `draft/game.js` をTHREE/DOMスタブで
   ヘッドレス実行して検証する（別実装のシミュレーションを新規に書くのは禁止）。
   スタブ・スモークテスト一式はscratchpad（セッション終了で消える）に都度作る
3. **反映**: 試作結果に問題がなければ、`draft/index.html` / `draft/game.js` /
   （必要なら）`draft/CLAUDE.md` の内容で、このフォルダ直下の同名ファイルを上書きする
4. **公開**: このフォルダで `git status` → 差分確認 → `git add` → `git commit` →
   `git push origin main`。pushすればGitHub Pagesの公開版に反映される
   （公開・push は破壊的操作に準じるため、必ずユーザーの明示的な指示があってから行う）

## ゲーム仕様
- 12頭立て。**5章×3レース=全15レース**から選択（相手馬も各レースの実際の出走馬）。
  章内の並び順は2026-07に調整済み（下記の順が現在の並び。data-race番号は0始まり通し）:
  - 第1章 伝説の幕開け: 1990安田記念(オグリキャップ) → 1994菊花賞(ナリタブライアン) →
    1992ジャパンC(トウカイテイオー)
  - 第2章 最強黄金世代: 1998日本ダービー(スペシャルウィーク) → 1998ジャパンC(エルコンドルパサー) →
    1999有馬記念(グラスワンダー)（順序は変更なし）
  - 第3章 英雄の衝撃: 2006天皇賞春(ディープインパクト) → 2004日本ダービー(キングカメハメハ) →
    2005有馬記念(ハーツクライ)。2005有馬記念はライバルのディープインパクトのadjを
    0.30→2.0に大幅強化(ゲーム内最高値・断トツ)、オッズは史実の1.3のまま
  - 第4章 気高き名牝たち: 2020ジャパンC(アーモンドアイ) → 2020宝塚記念(クロノジェネシス) →
    2020スプリンターズS(グランアレグリア)
  - 第5章 新時代の覇者: 2021皐月賞(エフフォーリア) → 2022天皇賞秋(イクイノックス) →
    2023有馬記念(ドウデュース)（順序は変更なし）
- 難易度カーブ(2026-07、章ごとではなく各章内の「レース位置」で統一): 各章とも
  1つ目のレースはプレイヤーadjを+0.05(易しめ)、2つ目は無補正(基準)、3つ目は
  プレイヤーadjを-0.10(かなり難しめ)。ただしプレイヤーadjの微調整だけでは
  体感できるほど難易度が変わらない(adj×0.4なので-0.10は-0.04m/sしかない)ため、
  各章3つ目のレースは「その章で最も手強いライバル」のadjを大幅強化する方式に変更:
  - 第1章3つ目(1992JC): ナチュラリズム 0.15→0.75
  - 第2章3つ目(1999有馬記念): スペシャルウィーク 0.22→0.9(一時1.2まで強化→やり過ぎで緩和)
  - 第3章3つ目(2005有馬記念): ディープインパクト 0.30→2.0(前述、唯一そのまま)
  - 第4章3つ目(2020スプリンターズS): ダノンスマッシュ 0.12→2.5(一時3.5→緩和)
    (1200mの短距離戦は道中の不利を挽回する猶予が短く、他レースより高いadjが必要)
  - 第5章3つ目(2023有馬記念): スターズオンアース 0.18→0.55(一時0.75→緩和)
- **adjブースト時は掛かり(kakari)の副作用に注意**: 差し/追込のライバルにadjを
  大きく盛ると、能力差で道中「前が開いた2番手以内」の状態に入りやすくなり、
  脚質不適性の掛かり判定(+0.40/秒、下記「折り合い」参照)を頻繁に踏んで
  終盤バテる・自滅するという逆効果が起きる(2026-07に発覚。ディープインパクトが
  勝率10%台に低迷していた主因はこれだった)。対策として馬データに
  `immuneKakari: true` を付けると、その馬だけ脚質に関わらず掛かり判定を免除できる
  (h.immuneKakariとして保持、suitedToLead判定に組み込み済み)。上記で強化した
  ナチュラリズム/スペシャルウィーク/ディープインパクト/スターズオンアース には
  immuneKakariを付与済み(ダノンスマッシュは元々「先行」で対象外のため不要)
- 2026-07: ディープインパクト以外の3つ目のレースは強化しすぎとのフィードバックを受け、
  adjを全体的に少し緩和(ナチュラリズム0.75→0.55/スペシャルウィーク1.2→0.9/
  ダノンスマッシュ3.5→2.5/スターズオンアース0.75→0.55)。ヘッドレス検証(各10走)での
  ライバル勝率: スペシャルウィーク5/10、スターズオンアース6/10、ナチュラリズム4/10、
  ダノンスマッシュ2/10
- **cruiseAdjMult / spurtMult / spurtAtCorner（2026-07追加）**: 「能力は保った
  まま道中で先頭に行かない」馬を作るための個別パラメータ。cruiseAdjMultはadjが
  道中の巡航速度(cruise)に効く割合(既定0.4、下げると道中の速さだけ抑えられ
  末脚(maxV)は変わらない)。spurtMultはスパート開始距離の倍率(既定1、上げると
  より早い地点から末脚を使い始める)。spurtAtCornerをtrueにすると、その馬の
  スパート開始距離を`REM_CORNER`(最後の3-4コーナー地点の残り距離、courseごとに
  自動計算)に固定できる(spurtMultより優先)。
  ディープインパクトはcruiseAdjMult0.15・spurtAtCorner trueを設定し、
  「2005有馬記念(中山内2500m=1.5周)の2周目3-4コーナーから進出開始」という
  史実の追い込みタイミングを再現。道中は中団〜後方に控え、直線で追い込む形。
  ただしこの調整には強いトレードオフがある: 道中の巡航を抑えるほど道中の遅れが
  蓄積し、スパート開始を遅らせるほど終盤で挽回できる距離が短くなるため、
  「道中一切先頭に行かない」ところまで抑えると勝率が大きく落ちる
  (ヘッドレス検証: cruiseAdjMult 0.05は勝率1/10まで低下)。
  1500m通過(2500m中60%地点)でほぼ先頭に行かない一方、勝率67%(12戦8勝)を
  確保する実測上の妥協点として、adjとcruiseAdjMultは常に
  「adj×cruiseAdjMult ≈ 0.3」の比を保つ(道中のcruise押し上げ分を一定に保つ)。
  2026-07: 「能力をさらに高めて(脚質は同じで)」の要望を受けadj 2.0→4.0・
  cruiseAdjMult 0.15→0.075に同時変更(比は据え置き)。maxVは常にadj×0.4を
  フルに受けるためスパート/末脚は単純に2倍相当に強化。道中の位置取り・勝率は
  据え置き(ヘッドレス検証で1500m通過順位・最終着順とも旧設定とほぼ同一と確認)。
  この比を保たずadjだけ上げる(≒cruiseAdjMultそのまま)と、あるライン(adj≈2.5〜)
  を超えたあたりから道中の巡航だけで先頭に立ってしまい「道中先頭に行かない」
  挙動が崩れることをヘッドレス検証で確認済み。adj/cruiseAdjMultをこの比率のまま
  さらに上げても(6.0/0.05まで確認)勝率は67%前後から伸びなかった
  (原因はスタミナ切れではなく渋滞だった。下記staminaEffMult/assertiveの項を参照)。
- **staminaEffMult（2026-07追加、このケースでは効果なしと判明）**: スタミナ
  消耗量に掛かる倍率(既定1、下げるほど消耗が緩くなる)。ディープの勝率停滞の
  原因が「終盤のスタミナ切れ」ではないか という仮説のもとに追加したが、
  ヘッドレス検証でstaminaEffMultを1.0→0.15まで下げても勝率は全く変化せず、
  仮説は否定された。原因はスタミナではなく後述のブロッキング(渋滞)だった。
  汎用パラメータとして残置(他の馬の調整で使う可能性があるため)
- **assertive（2026-07追加、勝率67%→95%を実現した本命の修正）**: 勝率停滞の
  真因をヘッドレス検証(敗因トライアルでの`blocked`フラグの発生率を追跡)で
  特定したところ、敗北したレースの90%以上でスパート区間中「他馬に前を
  塞がれて減速していた(blocked=true)」ことが判明。スタミナでも実力でもなく
  「渋滞に阻まれて追い込みきれない」ことが真因だった。assertiveをtrueにすると
  (1) sideBlockedの車線変更許可を自身のスパート区間中も緩和し、
  (2) 追加でブロック時の速度上限capVを自身のスパート区間中+0.8m/s引き上げ、
  「隙間を割って前に出る」ことを許可する。(1)だけでは効果薄く(blocked率は
  ほぼ変わらず)、(2)を加えて初めて詰まりを実質的に解消できた。
  ディープインパクトにassertive: trueを設定した結果、ヘッドレス検証で
  勝率67%(12戦8勝)→100%(12戦12勝、seed99000+t)、さらに別seedバッチでも
  95%(20戦19勝、seed500000+t*777)を確認。1500m通過時点の順位は
  「2,9,12,1,2,1,2,7,10,5,3,6,11,6,5,2,10,2,5」など道中は中団〜後方に
  留まっており、「道中先頭に行かない」挙動は維持されたまま勝率のみ改善された。
  2026-07: 「勝率が高すぎる、8.5割ぐらいにしてほしい」との要望を受け、
  adj 4.0→3.0・cruiseAdjMult 0.075→0.1に同時変更(比は0.3のまま据え置き、
  assertive/immuneKakari/spurtAtCorner等の他設定は変更なし)。比を保ったまま
  adjだけ下げるスイープをヘッドレス検証(各20戦、seed500000+t*777)した結果、
  adj=3.0/cruiseAdjMult=0.1で勝率85%(20戦17勝)と目標にほぼ一致したため
  これを採用(参考: adj=3.4は95%、adj=2.6も85%、adj=2.2は75%)
- レースのアンロック機能は無し。全15レース最初から自由に選択可能
- 脚質: 大逃げ/逃げ/先行/差し/追込。レースごとに速度補正(spdAdj)。
  消耗率(drainK)・初期スタミナは距離から自動スケーリング（後述）
- 「勝ち方デモ」機能は削除済み（2026-07）。demoBtn/demoTip/demoControl等は存在しない

## 距離ベースの自動スケーリング（initRace）
- `RACE.drainK = (720 / RACE.dist) * (RACE.drainMul || 1)` — 1200〜3200mで
  自動計算。dist×drainK≈720で統一（宝塚記念のみ `drainMul: 1.15` で重馬場を上乗せ→828）
- 初期スタミナ `staminaMax = 100 + (dist/1600)*12` で自動スケーリング
  （1200m→109、1600m→112、3200m→124）。h.staminaMaxとして各馬が保持
- バテ閾値(10%)・スタミナキック閾値(25%)・HUD低スタミナ警告(20%)は
  すべて`h.staminaMax`基準の相対値に統一済み（固定値ではない）

## ゲームメカニクス
- 集団収束(chase): 未ゴール先頭からの距離×0.02(最大+2.2m/s)を全馬の目標速度に加算
  (道中は0.6倍)。馬群が千切れず、終いは後方から飛んでくる。バテ下限は16.0+adj
- 最初の100mは全馬(プレイヤー含む)ゲート順の隊形のまま直進（密集回避。進路変更不可）
- AI進路: 100m以降は各馬固有の巡航レーン(cruiseLane, 枠順ベースで3列に自然分散)へ寄せる。
  詰まったら無理せず、1.5秒以上詰まると外の列へ(まくり)移って道中はその列を維持。
  まくり/詰まり時の外出しキャップはdrift 10・spurt中12.4→11に調整済み（外へ行き過ぎ防止、
  2026-07: 前回広げすぎたレーン幅の反動でここだけ元の値に戻した）
- スパートでは各馬に割当てた攻め進路(atkLane, 0.8〜9.8の11段階)に分散 → 直線でばらける
- ブロック時の減速(車間に応じた3段階のbrake): 車間1.6m未満3 / 2.6m未満2.2 /
  それ以上1.6 m/s^2。capVも「同速まで」に単純化(以前の0.97倍の追い討ちは廃止)。
  2026-07修正の経緯: 元は最も近い段が30m/s^2で1フレームで最大-1.0m/sの急停止が
  発生していたため9まで緩和したが、「横から急に壁が現れて減速する」感覚が
  まだ残るとの指摘があり、9→3にさらに緩和(ヘッドレス検証: 0.1m/s以上の
  減速イベントが5走14件・最大-0.13m/s程度まで低減。原因はブロック対象が
  車線変更で新規に真横〜斜め前に現れた瞬間に発生するもので、chase(集団収束)
  ボーナスの消失などは無関係と確認済み)
- 検証: draft/game.jsを実物のままTHREE/DOMスタブで無頭実行し結果テーブルを解析する
  (バランス検証は必ずこれで行うこと。別実装シミュは禁止。スタブはscratchpadに都度作る)

### 脚質バランス（2026-07再設計: 局面限定ボーナス方式）
高adjの差し・追込馬が能力差だけで道中フラフラ先頭に出てしまい、脚質に合わず
掛かって自滅する問題を解消するため、**cruise/maxVは全脚質共通の基礎値**
(cruise16.3・maxV18.15)に統一。脚質の強みは局面限定のボーナスに集約した:
- `early`（序盤〜400mの目標速度上乗せ、隊列形成用）: 大逃げ+1.1/逃げ+0.55/
  先行+0.3/差し-0.2/追込-0.35 — 従来通り、フラット化した基礎値の上に乗る分
  相対的な効果は強まっている
- `EARLY_ACCEL`（最初300mだけ加速度を強化。前で脚質に見合った位置を確保しやすくする）:
  大逃げ・逃げ×1.45、先行×1.2（差し・追込は補正なし）
- `LATE_KICK`（最後400mだけ最高速度に加算。末脚を局面限定で強化）:
  差し+0.35、追込+0.4（前脚質は補正なし）
- スパート開始距離(spurt)は従来通り脚質ごとに保持（大逃げ700m/逃げ・追込650m/
  先行640m/差し660m）。stamKick(残スタミナ変換、最大+1.0)は不変
- 勝利脚質分布はAI総当り検証(45走)で 差し20・先行10・大逃げ8・逃げ4・追込3。
  差し優性は維持しつつ前脚質合計と後脚質合計はほぼ互角(前22:後23)でバランス確認済み

### 折り合い（h.kakari、全馬対象）
残り900mまで管理が必要。「掛かり」上昇要因:
- 追う+0.25(序盤450mの位置取りはフリー、さらに最初の300mは掛かり自体が進行しない)
- ムチ+0.28
- 手綱を引く(↓)+0.38(追うより悪化・ハミを噛む、プレイヤーのみ)
- 詰まる+0.30(AIは+0.15、AI騎手は捌くのが上手い)
- 前が開いている(near.coverなし)時:
  - プレイヤー: 常に+0.16
  - AI・大逃げ/逃げ/先行、または`h.immuneKakari`が立っている馬: 追加なし
    （先頭適性がある、または個別に免除された馬は前が開いても平気）
  - AI・差し/追込(かつimmuneKakariなし)で**実際の順位が2番手以内**: +0.40
    （脚質に合わない先頭進出で本来の競馬ができず不安定になる、という設計）
  - AI・差し/追込で3番手以下（ただ前に一瞬空きがあるだけの中団）: +0.16
    （2026-07修正: 以前は順位を問わず「前が開いている」だけで+0.40が付き、
    差し・追込がほぼ全レースで掛かりゲージ最大に張り付く不具合があった。
    rankOf(h)<=2の判定を追加し「本当に先頭付近に出た時だけ」大きく掛かるよう修正）
0.5超で抑えが利かず消耗増(1.0で+80%)。解消手段は壁(前方12m以内の馬)+スリップ+
自然減衰のみ。AIの自然減衰は-0.20(プレイヤー-0.06)

## スタミナ・難易度
- スタミナ: 回復なし(使ったら戻らない)。序盤500m以内のハイペース消耗1.35倍 /
  スリップ0.6倍 / 壁アリ(cover)0.8倍 / staminaMaxの10%未満からなだらかに失速、
  0%で下限(16.0+adj)に達する。↓は消耗をほぼ止めるだけで回復はしない
- 難易度(2026-07): AIオフセット-0.03（プレイヤーへの特別なバフは無し）。
  プレイヤーpushV18.4・whipCap20.4（PLAYER定数の素の値）。
  プレイヤーの優位性は各レースのRACES.player.adj（史実の強さ、×0.4換算で
  cruiseV/pushV/easeV/whipCapに加算）のみに集約している
- ムチ: スタミナ2消費で1.5秒+1.2m/s、クールダウン1.2秒（回数無制限）

## カメラ・視点（2026-07: 左右確認を追加）
- 通常時: 前方視点（自馬の首・耳・ムチが見える）
- V押下: 後方確認（残り30m地点を注視、fp非表示）
- **C押下: 左確認**、**B押下: 右確認**（横方向ベクトル p.ox,p.oz を使い、

## その他の仕様
- HUDに走破タイム表示。先頭馬の1000m通過でペース判定(ハイ/ミドル/スロー)を表示
- レース選択→出馬表画面(枠番/馬番/脚質/単勝オッズ/人気)→ゲートインの流れ。
  オッズ・史実性の高い出走メンバーはRACESデータに直接保持。相手馬にcoat/silkが
  未指定の場合は枠順からデフォルトパレット(SILKS_DEF/COATS_DEF)を自動割当て
- ペース判定窓は各レースのpace:[lo,hi]。ペースの帰結(前後対称・プレイヤーも対象):
  1000m通過時点の順位で5番手以内=前群/6番手以降=後群を判定し、以後paceMulとして
  消耗率に掛かる。ハイペース→前群+12%消耗・後群-12%消耗。スローペース→逆
- レースごとにpaceBias(±0.3m/s)でAI全体のペースが振れ、展開が毎回変わる
- 着差圧縮: 馬ごと能力差はadj×0.4+乱数±0.05に縮小
- レーン幅は計+2.8m(馬4頭分、2026-07に+1.4mずつ2回拡張)に拡大: 外ラチ(17.6)/
  標識(19.6)/PLAYER.maxLane(15.0)/重なり解消クランプ(15.3)。ゴールライン
  (PlaneGeometry幅19.3・中心レーン8.65)も拡張後の全幅に合わせて再計算済み
  (2026-07: 前回のレーン拡張時に更新し忘れていたのを修正)。枠順ごとのゲート間隔
  (1.15m)や衝突判定の閾値(0.7/1.2/1.3m)、AIのdrift・詰まり時の外出しキャップ
  (10/11、前回の反動で据え置き済み)は馬の挙動側の値なので変更していない
- AIは道中に中弛み(ペースの波)があり馬群が伸縮する。HUDにポジション状態表示
  (スリップ中/詰まった/行きたがる)
- 最初の500m以降は真横(前後2.6m・横1.2以内)に馬がいると進路変更不可(すり抜け禁止)。
  2026-07: 最終直線(rem<=GOAL_MOD)に入ったらこの判定を(前後1.8m・横0.85)に緩和。
  完全にすり抜けられるわけではないが、直線での進路取り・追い抜きがしやすくなる
- スリップストリーム（前の馬の直後につくと消耗-40%）、進路ブロックあり

## 操作
- ↑/W: 追う（加速・スタミナ消費） ↓/S: 抑える（消耗をほぼ止める。回復はしない）
- ←→/A・D: 進路変更（内・外）
- Space: ムチ（1.5秒加速、スタミナ消費）
- V: 後方確認 / C: 左確認 / B: 右確認（いずれも押している間）
